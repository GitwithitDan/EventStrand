.rcal is an open file format for encoding recurring real-world events. It is designed to be simple to publish, simple to parse, and expressive enough to describe how venues, people, and communities actually operate — including weekly rhythms, monthly patterns, seasonal windows, exceptions, and one-off events.

The format is maintained by EventStrand but is not owned by it. Anyone can read, write, parse, or build tools around .rcal files without permission, license, or fee.

venue.rcal v0.1 
// Minimal valid .rcal file
{
  "rcal": "0.1",
  "meta": {
    "title": "The Rusty Nail",
    "type": "venue",
    "timezone": "America/Chicago",
    "live": true
  },
  "events": [
    {
      "id": "friday-jazz",
      "name": "Friday Jazz Night",
      "location": "inherits meta.location",
      // Override per-strand if event is at a different address
      "vibe_tags": [
        "mellow", "social"
      ],
      "recurrence": [
        {
          "pattern": "weekly",
          "every": 1,
          "days": ["friday"],
          "time_start": "21:00"
        }
      ]
    }
  ]
}
The .rcal Format

Powered by an
open format

Every EventStrand strand is backed by a .rcal file — an open JSON format anyone can read, implement, or build on. No license, no fees. The platform is EventStrand. The format is yours.

📅

Every recurrence pattern

Daily, weekly, bi-weekly, monthly by date or weekday, annual, seasonal windows, custom intervals. Stack multiple rules per event — because your favorite bar's happy hour runs weekdays and has extended Saturday hours, and those are two different rules.

🔁

Always up to date

Live strands are hosted on EventStrand and update automatically. When a publisher edits their strand — adds a show, cancels a night, changes a time — every subscriber's portal reflects it immediately. No rescanning, no re-downloading.

🎭

Vibe tagging built in

Mellow. Restless. Curious. Celebratory. Every event carries an emotional fingerprint. When you open the portal in a specific headspace, the filter already knows what to show you.

⚡

Exceptions and modifications

Closed for Christmas. Moving locations in March. Cancelled last month but back next week. Real venues are messy — .rcal handles all of it without breaking the thread.

🌍

Works at any scale

A dive bar with three weekly nights. A city arts council with 200 events. A global strand for Ramadan, synchronized across timezones. Same format. Same fifteen fields.

example.rcal Open Format 
// Multi-rule recurrence example
{
  "rcal": "0.1",
  "meta": {
    "title": "The Rusty Nail",
    "type": "venue",
    "location": {
      "address": "1234 N Milwaukee Ave",
      "city": "Chicago",
      "state": "IL",
      "country": "US",
      "lat": 41.9214,
      "lng": -87.7078
    },
    "timezone": "America/Chicago",
    "live": true,
    "color": "#C0392B"
  },
  "events": [
    {
      "id": "happy-hour",
      "name": "Happy Hour",
      "vibe_tags": [
        "spontaneous", "social"
      ],
      "price": "free_entry",
      "lead_time_days": 0,
      "recurrence": [
        {
          // Weekday happy hour
          "pattern": "weekly",
          "days": ["weekday"],
          "time_start": "16:00",
          "time_end": "19:00"
        },
        {
          // Weekend extended hours
          "pattern": "weekly",
          "days": ["saturday"],
          "time_start": "14:00",
          "time_end": "18:00"
        }
      ],
      "exceptions": [
        {
          "type": "skip",
          "date": "2026-12-25",
          "note": "Closed Christmas"
        }
      ]
    },
    {
      "id": "open-mic",
      "name": "Open Mic Night",
      "vibe_tags": [
        "curious", "social"
      ],
      "recurrence": [
        {
          // Second Thursday + Last Saturday
          "pattern": "monthly_week",
          "month_week": "second",
          "days": ["thursday"],
          "time_start": "20:00"
        },
        {
          "pattern": "monthly_week",
          "month_week": "last",
          "days": ["saturday"],
          "time_start": "21:00"
        }
      ]
    }
  ]
}

For Venues & Creators

Posting your strand to your site

Once you've published a strand, post the QR code or link on your website so people can find it. Pick your platform below for step-by-step instructions.

Squarespace

Squarespace lets you add images and buttons through its block editor. You can post your strand as a QR image, a text link, or both.

1
Open your site in Squarespace and navigate to the page you want to post on — your homepage, events page, or a dedicated "Find Us" page.
2
To post the QR image: Click Add Block, choose Image, and upload your strand QR code PNG (download it from your EventStrand dashboard → QR Code).
3
To post a text link: Click Add Block, choose Text, type something like "Subscribe to our schedule," highlight it, click the link icon, and paste your strand URL (from dashboard → Copy Strand Link).
4
Optionally add a caption below the QR image: "Scan to follow our events on EventStrand."
5
Click Save in the top-right, then Publish. Copy your page URL and paste it into the Directory verification field in EventStrand.

Wix

Wix uses a drag-and-drop editor. You can add images, buttons, and text anywhere on the page.

1
Open the Wix Editor and click into the page where you want to post your strand.
2
To post the QR image: Click the blue + button in the left panel, select Image, and upload your strand QR code PNG.
3
Resize and position the QR image. Right-click the image → Link → paste your strand URL so tapping the image on mobile also works.
4
To post a text link: Add a Text element, type your call-to-action, highlight the text, and use the link tool to paste your strand URL.
5
Click Publish in the top-right. Use the published page URL for directory verification.

WordPress

Works with both the Block Editor (Gutenberg) and Classic Editor.

1
In your WordPress admin, go to Pages (or Posts) and open the page you want to edit — or create a new one called "Events" or "Find Us."
2
Block Editor: Click the + icon to add a block. Choose Image to upload your strand QR code, or Paragraph then highlight text and use the link icon to paste your strand URL.
3
Classic Editor: Click the image icon in the toolbar to upload your QR code, or highlight text and click the link icon to paste your strand URL.
4
Add alt text to your QR image: "EventStrand QR code — scan to follow our schedule." This helps with accessibility and confirms the image purpose.
5
Click Update or Publish. Use the page's public URL in the directory verification field.

Shopify

Best placed on your homepage via a custom section, or on a dedicated page. If you host events in-store, the homepage or an "Events" page works well.

1
In your Shopify admin, go to Online Store → Pages and click Add page (or open an existing one like your homepage).
2
In the page editor, click the image icon in the content area to upload your strand QR code PNG.
3
To add a text link: type your call-to-action, highlight it, click the link icon, and paste your strand URL. Set it to open in a new tab.
4
For homepage sections: go to Online Store → Themes → Customize, find a section that accepts images or rich text, and add your QR image or link there.
5
Click Save. Use your published page URL for directory verification.

Webflow

Webflow gives you complete layout control. The easiest approach is adding an Image element with a link wrapper, or a Rich Text block.

1
Open your Webflow Designer and click into the page you want to edit.
2
QR image: Drag an Image element onto the canvas. In the Settings panel on the right, upload your strand QR code PNG.
3
Wrap the image in a Link Block (drag Link Block first, then drop the Image inside it). Set the link's URL to your strand URL so mobile taps work.
4
Text link: Add a Text Link element, set its text (e.g., "Follow our event schedule"), and paste your strand URL in the Link Settings panel.
5
Publish your site. Copy the published page URL and paste it into the directory verification field.

GoDaddy Website Builder

GoDaddy's builder uses a section-based editor. Add an image section for your QR code or use a text section for a link.

1
Log into GoDaddy and open your website editor. Navigate to the page you want to update.
2
Click the + icon between sections to add a new section. Choose Image & Text or a standalone Image section.
3
Click the image placeholder and upload your strand QR code PNG from your device.
4
To add a text link: click into the text area of the section, highlight your call-to-action text, and use the link icon to paste your strand URL.
5
Click Publish in the top-right corner. Copy your live page URL and use it for directory verification.

Google Business Profile

Google Business doesn't support clickable links in posts, but you can post your QR code as a photo and include the strand URL in the post text. Use your website URL for directory verification instead.

1
Go to business.google.com and select your business profile.
2
Click Add update (or Create post). Write your post text including your strand URL — for example: "Follow our weekly schedule at eventstrand.com/s/yourhandle/strandid"
3
Click the photo icon to attach your strand QR code image to the post. This lets customers scan it directly from your Google listing.
4
Also go to Edit profile → Website and add your events page URL (or the strand link directly) so it appears permanently on your profile.
5
For directory verification, use your business website URL (where you've also posted the link) rather than your Google profile URL.

Linktree / Beacons

Link-in-bio tools are the fastest option. Add your strand as a button — when your followers tap your bio link, they'll see it immediately.

1
Log into your Linktree or Beacons dashboard.
2
Linktree: Click Add link, paste your strand URL into the URL field, and give it a title like "📅 Follow our weekly schedule."
3
Beacons: Click Add block, choose Link, paste your strand URL, and enter a display name.
4
Drag the link to near the top of your list so it's visible without scrolling.
5
Save changes. Use your Linktree or Beacons profile URL (e.g., linktr.ee/yourvenue) as the verification URL in EventStrand.

parse-example.js JavaScript 
// Fetch and parse a live .rcal strand
import { parseRcal, getUpcoming } from '@eventstrand/rcal-js';

const res = await fetch('https://eventstrand.com/s/rusty-nail');
const strand = await parseRcal(await res.text());

// Get all events in the next 7 days
const upcoming = getUpcoming(strand, {
  from: new Date(),
  days: 7,
  vibeFilter: ['mellow', 'social']
});

upcoming.forEach(event => {
  console.log(
    `${event.name} — ${event.date} at ${event.time_start}`
  );
});

// Output:
// Friday Jazz Night — 2026-03-14 at 21:00
// Happy Hour — 2026-03-16 at 16:00
// Open Mic Night — 2026-03-19 at 20:00
